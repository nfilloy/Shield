1. VISION GENERAL
================================================================================

1.1 Que es este proyecto?
-------------------------
Un sistema de deteccion de amenazas que identifica:
- Smishing: Phishing a traves de SMS (mensajes de texto fraudulentos)
- Phishing: Correos electronicos fraudulentos

El sistema utiliza Machine Learning y Procesamiento de Lenguaje Natural (NLP)
para clasificar mensajes como legitimos o maliciosos.

3.2 Extraccion de caracteristicas
---------------------------------

El sistema combina DOS tipos de features:

A) TF-IDF (Term Frequency - Inverse Document Frequency)
   - Word n-grams: (1,3) -> unigrams, bigrams, trigrams
   - Char n-grams: (2,5) -> captura patrones ortograficos
   - max_features: 5,000 por tipo
   - Total TF-IDF: ~10,000 features

B) Features estructurales/URL (src/features/sms_features.py)

+------------+-----------------------------------------------------------------------------+
| Categoria  | Features extraidas                                                          |
+------------+-----------------------------------------------------------------------------+
| URLs       | url_count, has_url, shortened_url_count, suspicious_tld_count, ip_url_count |
| Keywords   | urgency_count, promo_count, financial_count, action_count                   |
| Caracteres | uppercase_ratio, digit_ratio, special_char_ratio, emoji_count               |
| Patrones   | has_verification_code, has_tracking_number, impersonation_indicator         |
+------------+-----------------------------------------------------------------------------+

Total features combinadas: ~10,044 (TF-IDF + 44 estructurales)

6.2 Flujo de prediccion
-----------------------

    class PhishingDetector:
        def predict(self, text, model_name, text_type='sms'):
            # 1. Preprocesar texto
            clean_text = self.preprocessor.preprocess(text)

            # 2. Extraer features del texto ORIGINAL (SMS o Email)
            if text_type == 'sms':
                features = self.sms_feature_extractor.extract_all_features(text)
                vec_key, char_key, svd_key, url_config_key = 'tfidf', 'char', 'svd', 'sms'
            else:
                features = self.email_feature_extractor.extract_all_features({
                    'body': text, 'subject': '', 'headers': {}
                })
                vec_key, char_key, svd_key, url_config_key = 'email_tfidf', 'email_char', None, 'email'

            # 3. Ajustar nombre de modelo para email
            if text_type == 'email' and not model_name.startswith('email_'):
                model_name = f'email_{model_name}'

            # 4. Fallback si faltan modelos/vectorizadores
            if model_name not in self.models or vec_key not in self.vectorizers:
                return self._rule_based_detection(text, features, text_type)

            # 5. Vectorizar texto preprocesado (word + char + SVD opcional)
            X_word = self.vectorizers[vec_key].transform([clean_text])
            X_char = self.vectorizers[char_key].transform([clean_text]) if self.vectorizers.get(char_key) is not None else None
            X_tfidf = sparse_hstack([X_word, X_char]) if X_char is not None else X_word
            if svd_key and self.vectorizers.get(svd_key) is not None:
                X_tfidf = self.vectorizers[svd_key].transform(X_tfidf)
            if hasattr(X_tfidf, 'toarray'):
                X_tfidf = X_tfidf.toarray()

            # 6. Combinar con URL/configuracion guardada
            if url_config_key in self.url_configs:
                feature_names = self.url_configs[url_config_key]['feature_names']
                X_url = np.array([[features.get(name, 0) for name in feature_names]])
                X = np.hstack([X_tfidf, X_url])
            else:
                X = X_tfidf

            # 7. Validar dimensionalidad y predecir
            model = self.models[model_name]
            expected_features = getattr(model, 'n_features_in_', None)
            if expected_features is not None and X.shape[1] != expected_features:
                return self._rule_based_detection(text, features, text_type)

            prediction = model.predict(X)[0]
            probability = model.predict_proba(X)[0]

            return {
                'prediction': int(prediction),      # 0 = legitimo, 1 = phishing
                'probability': float(probability[1]) * 100,   # % de ser phishing
                'confidence': max(probability) * 100,
                'features': features,
                'clean_text': clean_text,
                'model_used': model_name
            }

7. EXPLICABILIDAD (XAI)
================================================================================

7.1 Por que XAI?
----------------

La explicabilidad permite entender POR QUE el modelo toma una decision,
no solo QUE decision toma. Esto es crucial para:
- Validar que el modelo aprende patrones correctos
- Generar confianza en el usuario
- Cumplir con requisitos de transparencia


7.2 Tecnicas implementadas
--------------------------

SHAP (SHapley Additive exPlanations)
- Basado en valores de Shapley de teoria de juegos
- Muestra la contribucion de cada feature a la prediccion
- TreeExplainer para XGBoost/Random Forest
- LinearExplainer para Logistic Regression

LIME (Local Interpretable Model-agnostic Explanations)
- Crea un modelo lineal local alrededor de la prediccion
- Funciona con cualquier modelo (model-agnostic)
- Resalta palabras que contribuyen a la clasificacion


7.3 Ejemplo de salida SHAP
--------------------------

Texto: "URGENTE: Tu cuenta sera suspendida. Verifica: bit.ly/xyz"

Features mas importantes:
+--------------------+--------------+-----------------------------------+
| Feature            | SHAP Value   | Interpretacion                    |
+--------------------+--------------+-----------------------------------+
| has_shortened_url  | +0.45        | URL acortada aumenta riesgo       |
| urgency_count      | +0.32        | Palabras de urgencia              |
| "verifica"         | +0.28        | Keyword de accion                 |
| "cuenta"           | +0.15        | Contexto financiero               |
+--------------------+--------------+-----------------------------------+
